2. NOTES ON FILES
======================================================================

- app.py: Flask backend. Serves UI/API. Manages users/prefs/Caddyfile. Triggers Caddy reload.
  - Uses `_find_matching_brace()` and `_remove_directive_block()` for proper Caddyfile parsing (handles nested braces, strings, comments).
  - `_add_log_to_site_blocks(content, global_start, global_end)`: Scans top-level blocks outside the global block; adds `log` directive to any site block that lacks one. Preserves indentation. Processes in reverse order to keep offsets valid.
  - `_configure_caddyfile_logging_internal()`: Two-step approach: (1) add/replace global `log { output stdout format json level INFO }` block, (2) add `log` to every site block that lacks one. This ensures Caddy actually emits access logs.
  - Secret key: auto-generated via `secrets.token_hex(32)` if FLASK_SECRET_KEY is not set or uses default insecure key.
  - Session cookies: HTTPOnly=True, SameSite=Lax.
  - `CADDY_CONFIG_FILE` reads from both `CADDY_CONFIG` and `CADDY_CONFIG_FILE` env vars (fallback chain).
  - Stats API: `/api/stats?period=7d&host=` and `/api/stats/hosts`. Backed by `stats_aggregator.py`. No longer reads raw logs on every request — uses SQLite aggregation.
  - `STATS_DB_PATH = APP_DATA_DIR / 'stats.db'` — initialized both at module level (for gunicorn) and in `__main__` (for dev server).
  - Error rate counts only 5xx (not 4xx).
  - `CADDY_ACCESS_LOG_FILE` default: `/var/log/caddy_panel/caddy_access.json.log` (persisted via Docker volume).
  - `GEOIP_DB_PATH` env var: path to GeoLite2-Country.mmdb (default: `APP_DATA_DIR/GeoLite2-Country.mmdb`). Auto-configured if file exists at startup.
  - `maxmindLicenseKey` preference: stored in preferences.json. Masked in GET API (`********XXXX`). Auto-downloads GeoIP DB on preference save if key provided and DB not present.
  - `POST /api/geoip/download`: manual trigger to download GeoIP DB using stored license key.
  - `GET /api/geoip/status`: returns `{geoip_available: bool}`.
  - `_try_geoip_download_and_configure(license_key)`: downloads GeoLite2-Country.mmdb from MaxMind, extracts, configures stats_aggregator.
- stats_aggregator.py: Incremental log processor + stats query engine.
  - SQLite DB with `PRAGMA journal_mode=WAL` for safe concurrent access.
  - Two tables: `hourly_stats` (retention 7d) and `daily_stats` (retention 365d). Schema: `(bucket, host, total, status_1xx..5xx, total_duration, total_size, error_count, top_paths JSON, top_uas JSON, top_countries JSON)`.
  - `_parse_ts(ts_value)`: Accepts both numeric (epoch) and string (RFC3339/ISO 8601) timestamps.
  - `process_new_logs()`: Extracts `request.remote_ip` (falls back to `X-Forwarded-For` / `X-Real-Ip`). Resolves to ISO country code via `geoip2` if available, else 'Unknown'. Stores in `top_countries`.
  - `configure_geoip(db_path)`: Optional. Loads MaxMind GeoLite2-Country.mmdb. If unavailable, country stats return 'Unknown'.
  - `_resolve_country(ip_str)`: Returns ISO 3166-1 alpha-2 code or 'Unknown'.
  - `is_geoip_available()`: Check if GeoIP DB is loaded.
  - `get_stats()`: Now includes `top_countries` (top 10 by count) and `geoip_available` (bool).
  - DB migration: on init, `ALTER TABLE ADD COLUMN top_countries` if column missing.
- Dockerfile: Multi-stage build. Installs Caddy binary, Python deps, app code. Defines `CADDY_ACCESS_LOG_FILE=/var/log/caddy_panel/caddy_access.json.log`. Creates and chowns `/var/log/caddy_panel`.
- docker-compose.yml: Deployment orchestration. Defines service, volumes (including `./caddy_logs:/var/log/caddy_panel` for persistent logs), env vars. `CADDY_ACCESS_LOG_FILE` explicitly set.
- docker/entrypoint.sh: Init script. No longer depends on `jq`. Creates default configs, manages permissions.
  - Auto-configures Caddyfile logging: always runs Python heredoc (no more grep-skip). Adds global log block + `log` directive to every site block.
  - Uses `python3 << 'PYEOF'` heredoc (single-quoted delimiter) to avoid bash double-quote escaping issues. Reads `CADDY_CONFIG_FILE` from environment via `os.environ`.
  - Creates `/var/log/caddy_panel` directory and chowns it.
  - GeoIP: if `MAXMIND_LICENSE_KEY` is set and GeoLite2-Country.mmdb not yet present, auto-downloads it to `APP_DATA_DIR/GeoLite2-Country.mmdb`. Exports `GEOIP_DB_PATH` env var.
- docker/supervisord.conf: Manages `caddy` (root) and `flaskapp` (appuser) processes.
  - Caddy runs WITHOUT `--resume` flag (uses Caddyfile on startup).
  - Caddy stdout → `/var/log/caddy_panel/caddy_access.json.log` (persisted via volume).
  - Single `user=root` for caddy (removed duplicate).
  - Flaskapp environment passes both `CADDY_CONFIG` and `CADDY_CONFIG_FILE`, plus `CADDY_ACCESS_LOG_FILE`.
- templates/index.html: XSS protection via `|tojson|replace('<\/', '<\\/')|safe` for initial data script.
- templates/login.html & setup.html: Apply stored theme from localStorage on page load.
- templates/stats.html: Full viewport dashboard with CSS Grid layout (no scroll). Period selector + host filter. KPI cards, status bar, country cards, world heatmap (D3+TopoJSON from CDN), timeseries chart (Chart.js). Top 5 countries with flag emojis + bar indicators. Graceful if GeoIP unavailable.
- static/script.js: Caddyfile parsing uses `findMatchingBrace()` (proper nested brace handling). Validates modal inputs. Stats button opens `/stats?host=X`.
- static/style.css: Dark theme styles for auth pages.
- caddyfile/Caddyfile: Example Caddyfile for initialization. Has global `log { output stdout format json level INFO }` (no `time_format rfc3339`).
- requirements.txt: Flask, Werkzeug, gunicorn.

3. KEY DESIGN DECISIONS
======================================================================

CADDY LOGGING — CRITICAL ARCHITECTURE:
- Caddy does NOT emit access logs for a site unless that site block has a `log` directive. The global `log { output stdout format json }` only configures the DEFAULT logger format/output — it does NOT automatically emit access logs.
- Therefore, both the global block AND each site block must have `log`. The entrypoint.sh and `_configure_caddyfile_logging_internal()` both ensure this.
- Caddy outputs JSON to stdout. Supervisord captures stdout → `/var/log/caddy_panel/caddy_access.json.log`.
- This file is persisted via Docker volume (`./caddy_logs:/var/log/caddy_panel`).
- IMPORTANT: `time_format rfc3339` in the Caddyfile global log config causes `ts` to be a STRING (e.g., `"2024-01-15T10:30:00Z"`). The stats_aggregator only processed numeric `ts`, so ALL entries were silently skipped → "No processable log entries found." The fix: removed `time_format rfc3339` from all Caddyfile templates AND added `_parse_ts()` to handle both formats.

HEREDOC FOR PYTHON IN ENTRYPOINT.SH:
- Must use `python3 << 'PYEOF'` (single-quoted delimiter). This tells bash to NOT process any escapes, variables, or backslashes inside the heredoc. Plain Python code can be written without bash double-quote escaping hell.
- Must read environment variables via `os.environ['CADDY_CONFIG_FILE']` instead of `'$CADDY_CONFIG_FILE'` since the heredoc doesn't expand bash variables.

DOCKER BUILD CACHE ISSUE:
- GitHub Actions workflow uses `cache-from: type=gha` which can serve stale Docker layers. If a file change doesn't seem to take effect in the container despite being committed, bust the cache with `no-cache: true` in the workflow, then re-enable cache once confirmed working.

COMPLETED_BUGFIXES (v1.4):
- CRITICAL: entrypoint.sh Python inline script had bash double-quote escaping hell — `python3 -c "..."` with nested backslashes produced `SyntaxError: unterminated string literal`. Fixed by switching to `python3 << 'PYEOF'` heredoc with single-quoted delimiter (no bash escaping) + `os.environ` for variables.
- CRITICAL: Caddy access logs not emitted despite global `log { format json }` config. Root cause: Caddy requires each site block to have a `log` directive to emit access logs; global config only sets default logger format. Fix: `_add_log_to_site_blocks()` in app.py + equivalent in entrypoint.sh Python heredoc.
- CRITICAL: `time_format rfc3339` in Caddyfile global log caused all entries to be silently skipped by stats_aggregator (`isinstance(ts, (int, float))` check fails on string timestamps). Fix: removed `time_format rfc3339` from all Caddyfile templates; added `_parse_ts()` to handle both numeric and RFC3339 timestamps.
- MODERATE: Stats chart "Requests per Hour" expanded vertically infinitely. Fix: changed `.chart-container` from `min-height: 300px` to `height: 350px`.
- MODERATE: Docker build cache (GHA) serving stale layers. Fix: temporarily set `no-cache: true` in workflow.
- MODERATE: Log file `/var/log/caddy_access.json.log` lost on container recreation. Fix: moved to `/var/log/caddy_panel/caddy_access.json.log` persisted via Docker volume.
- MODERATE: `_add_log_to_site_blocks()` had unterminated docstring (two `"` instead of three `"`) and broken result-building logic. Rewrote with cleaner reverse-order offset approach.
