2. NOTES ON FILES
======================================================================

- app.py: Flask backend. Serves UI/API. Manages users/prefs/Caddyfile. Triggers Caddy reload.
  - Preferences include: theme, caddyfilePath, globalAdminEmail, maxmindAccountId, maxmindLicenseKey, defaultAuthentikEnabled, defaultAuthentikOutpostUrl, defaultAuthentikUri, defaultAuthentikCopyHeaders, defaultAuthentikTrustedProxies, defaultSkipTlsVerify.
  - Credentials masking: GET /api/preferences masks maxmindAccountId (****XXX) and maxmindLicenseKey (********XXXX). POST detects masked values and keeps existing ones.
  - `_try_geoip_download_and_configure(account_id, license_key)`: Strategy 1: runs `geoipupdate` binary (most reliable). Strategy 2: HTTP download from MaxMind API with Basic Auth. Returns (success, message).
  - `POST /api/geoip/download`: triggers download using stored credentials.
  - `POST /api/geoip/upload`: accepts .mmdb file upload, saves to GEOIP_DB_PATH, configures stats_aggregator.
  - `GET /api/geoip/status`: returns {geoip_available: bool}.
  - `GET /api/geoip/test`: tests credentials without downloading full DB.
  - On preference save: if maxmindAccountId + maxmindLicenseKey provided and GeoIP DB not present, auto-downloads.
  - `_find_matching_brace()`, `_remove_directive_block()`, `_add_log_to_site_blocks()`, `_configure_caddyfile_logging_internal()`: Caddyfile parsing helpers.

- stats_aggregator.py: Incremental log processor + stats query engine.
  - SQLite DB with `PRAGMA journal_mode=WAL`. Two tables: hourly_stats (7d retention) and daily_stats (365d retention).
  - Schema columns: bucket, host, total, status_1xx..5xx, total_duration, total_size, error_count, top_paths JSON, top_uas JSON, top_countries JSON.
  - `process_new_logs()`: extracts `request.remote_ip` (fallback: X-Forwarded-For, X-Real-Ip). Resolves to ISO country code via `geoip2` if available, else 'Unknown'.
  - `configure_geoip(db_path)`: loads MaxMind GeoLite2-Country.mmdb. Graceful if unavailable.
  - `_resolve_country(ip_str)`: returns ISO 3166-1 alpha-2 code or 'Unknown'.
  - `is_geoip_available()`: checks if GeoIP reader is loaded.
  - `get_stats()`: returns `top_countries` (top 10) + `geoip_available` (bool).
  - DB migration: on init, `ALTER TABLE ADD COLUMN top_countries` if missing.

- Dockerfile: Multi-stage build.
  - Stage 1 (caddy_builder): downloads Caddy binary.
  - Stage 2 (geoip_builder): downloads `geoipupdate` binary from MaxMind GitHub releases (v7.0.1). Supports amd64 + arm64.
  - Stage 3 (final): python:3.10-slim-bullseye. Installs supervisor + curl. Copies Caddy + geoipupdate binaries from builder stages.
  - Installs `geoip2` via pip (from requirements.txt).
  - `GEOIP_DB_PATH` env var defaults to `/app_data/GeoLite2-Country.mmdb`.

- docker/entrypoint.sh:
  - Auto-configures Caddyfile logging via Python heredoc.
  - GeoIP download: tries `geoipupdate` first (if available + credentials set), then falls back to `curl` with Basic Auth. Uses `MAXMIND_ACCOUNT_ID` + `MAXMIND_LICENSE_KEY` env vars.
  - Generates `/app_data/GeoIP.conf` for `geoipupdate` on the fly.

- docker-compose.yml: Documents `MAXMIND_ACCOUNT_ID` and `MAXMIND_LICENSE_KEY` env vars (commented out).

- templates/index.html: Preferences page has a "GeoIP / Statistics" section with:
  - MaxMind Account ID input
  - MaxMind License Key input (password type, masked in API)
  - "Test" button (tests credentials via `/api/geoip/test`)
  - "Download" button (downloads via `/api/geoip/download` using geoipupdate then HTTP fallback)
  - "Upload manually" file input (uploads .mmdb via `/api/geoip/upload`)
  - Link to MaxMind signup page

- templates/stats.html: Full-viewport CSS Grid dashboard (no scroll). Uses CSS custom properties (--accent, --bg-card, etc.) for theming.
  - D3 + TopoJSON world map heatmap from CDN (world-atlas@2).
  - Top 5 countries with flag emojis.
  - KPI cards, status bar, list cards with overflow-y auto.
  - Chart.js timeseries.
  - Stats API: `/api/stats?period=7d&host=` returns `top_countries` + `geoip_available`.

- static/style.css: Redesigned theme system using CSS custom properties.
  - 10 themes (5 light + 5 dark): only accent color changes per theme. Base is neutral gray for all.
  - Variables: --accent, --accent-hover, --accent-contrast, --bg-page, --bg-card, --bg-elevated, --bg-input, --text-primary, --text-secondary, --text-heading, --border, --border-strong, --shadow.
  - Light themes: white/gray base. Dark themes: dark gray base.

- templates/login.html & setup.html: Use CSS variables from style.css. Clean centered auth forms.

- requirements.txt: Flask, Werkzeug, gunicorn, geoip2>=4.0.0.

3. KEY DESIGN DECISIONS
======================================================================

CADDY LOGGING:
- Both global `log { output stdout format json }` AND `log` in each site block are required.
- No `time_format rfc3339` (causes string timestamps that break stats_aggregator).
- `_parse_ts()` handles both numeric and string timestamps.

GEOIP:
- Three ways to get the GeoLite2-Country.mmdb database:
  1. UI Preferences → enter Account ID + License Key → "Download" (uses `geoipupdate` then HTTP fallback)
  2. UI Preferences → "Upload manually" .mmdb file
  3. Docker env vars `MAXMIND_ACCOUNT_ID` + `MAXMIND_LICENSE_KEY` → auto-download at startup
- `geoipupdate` is the primary download method (uses MaxMind's own protocol). HTTP Basic Auth is fallback.
- If no GeoIP DB is available, country stats show "Unknown" and the map shows "No geographic data".
- Credentials are masked in the GET /api/preferences response. Masked values (****XXX / ********XXXX) are preserved on POST.

THEME SYSTEM:
- All themes use the same neutral gray base. Only accent colors differ (buttons, active tabs, links, KPI values).
- CSS custom properties (--accent, --bg-card, etc.) defined per theme class on body.
- Light themes: white cards, dark text. Dark themes: dark cards, light text.

STATS LAYOUT:
- Full viewport (100vh, overflow hidden) CSS Grid: 3 columns × 3 rows.
- KPI cards (4) in row 1 left, Countries + world map in row 1-2 right.
- Status, Hosts, Paths, UAs in scrollable list cards.
- Chart in bottom-right cell.

DOCKER BUILD:
- Multi-stage: caddy_builder → geoip_builder → final.
- geoipupdate binary downloaded from GitHub releases (supports amd64 + arm64).
- `python:3.10-slim-bullseye` base image (Debian 11, proven).

4. PROJECT STATUS
======================================================================

ACTIVE_BUGS / KNOWN_LIMITATIONS:
- No CSRF protection on POST endpoints.
- No login rate-limiting.
- No password change feature.
- Single-user system.
- MaxMind HTTP API download may return 401 even with valid credentials (geoipupdate protocol is more reliable).
- BASE_DIR for file browsing is CWD (includes app source code).

ROADMAP:
- Add CSRF protection.
- Add login rate-limiting.
- Add password change feature.
- Restrict `/api/browse` and `/api/readfile` to specific directories.

COMPLETED_FEATURES:
- Containerized architecture (Docker/Compose).
- Process management (Supervisor).
- Automatic initialization (entrypoint.sh).
- Data persistence (volumes).
- User authentication.
- Caddyfile management UI (Configurator + Raw Editor).
- Automatic saving and reloading of Caddy.
- Preferences management UI (including GeoIP section).
- Statistics page with SQLite-backed aggregation (v1.3).
- Statistics dashboard: full-viewport CSS Grid layout, no-scroll. CSS variable theming.
- GeoIP country tracking: top 5 countries + world heatmap (D3+TopoJSON). Upload or download GeoLite2-Country.mmdb.
- Caddy log configuration wizard.
- Multi-architecture Docker build (amd64 + arm64).
- Automatic JSON logging configuration for all sites.

- MODERATE: Themes were overly colorful (every element colored per theme). Fix: complete redesign using CSS custom properties. Neutral gray base, only accent colors vary.
- IMPORTANT: GeoIP download via MaxMind API returned 401 with valid credentials. Fix: added `geoipupdate` as primary download method (installed from GitHub releases in Docker). Added manual .mmdb upload fallback. Added Test Connection button.
- IMPORTANT: `maxmindLicenseKey` alone insufficient — MaxMind requires Account ID + License Key for API access. Fix: added `maxmindAccountId` preference field. Both fields required for download.
